# FBA Inventory Decision Web

面向运营人员的静态网页工具，用于分析 Amazon FBA 库存、库龄、仓储费、清算预计净回收与移除费。

## 隐私设计

- 公开仓库只包含脱敏演示数据。
- 运营报告通过浏览器本地读取，不会上传到 GitHub 或其他服务器。
- 页面刷新后，已选择的报告和分析结果会被清除。

## 支持的文件

- FBA 库存报告
- FBA 库龄报告
- 月度仓储收费报告
- 销售佣金预览报告
- 所有商品报告
- 成本补充表（采购成本、FBA正常销售配送费、头程三项占售价比例）
- 包含兼容字段的 CSV / TSV

支持 XLSX、XLS、XLTX、CSV、TSV，可分多次选择文件且不会覆盖已选文件。页面可下载成本补充模板，也可统一填写采购成本、FBA配送费和头程占当前售价的比例；成本表中的 SKU 比例优先。当前费率表覆盖 US、CA、UK、DE，页面会显示规则版本和数据缺口。

页面按最新详细库龄快照展示 `0-180`、`181-210`、`211-240`、`241-270`、`271-300`、`301-330`、`331-365`、`366-455`、`456+` 各区间的库存件数和 SKU 数，并根据站点标记是否进入收费区间。

“现在清算”按清算毛回收、Amazon 清算费用、清算现金回收、采购成本、头程逐项拆分，最终展示扣除成本与头程后的清算净回收。继续销售方案也扣除同一批库存的采购成本与头程，保证两个方案使用一致口径。移除方案同时展示正数的 Amazon 移除费，以及负数表示的移除总损失（采购成本、头程和 Amazon 移除费）；缺少移除后回收价值时不参与最终收益比较。

清算和移除的测算数量仅使用当前站点已进入长期仓储计费区间的库存。预计冗余库存仍用于风险识别，但不参与清算净回收、Amazon 移除费和移除总损失计算。

页面会列出缺少销售或成本口径的具体 SKU 和缺失字段。长期仓储计费库存、预计冗余库存和高风险 SKU 三个指标可点击筛选下方 SKU 明细，并下载当前筛选结果；少量缺数据 SKU 可临时排除，仅对其余 SKU 生成处置建议。执行清单可按搜索、风险、建议动作和经济影响筛选排序，并下载当前结果。

广告清货是独立的可选路径。填写清货售价占现价比例、广告费占清货销售额比例和清货销量占当前30日销量比例后，系统会把广告清货与正常销售、立即清算放在同一口径比较；成本补充表可通过 `clearance-price-rate`、`advertising-cost-rate` 和 `clearance-sales-30` 按 SKU 覆盖默认值，其中 `clearance-sales-30` 填预计 30 日销售件数。页面和 CSV 同时展示广告清货预计结果、广告投入以及每件最大可承受广告费。
